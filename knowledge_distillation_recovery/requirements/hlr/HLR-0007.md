## Description

When the YAML's `save_every_n_steps` is non-zero, kdr SHALL write a partial checkpoint to a directory named `kdr_{mode}_partial_step{N}/` under the artifacts root, then synchronously upload that directory to a private HF Hub model repo via `huggingface_hub.upload_folder`. Save and upload SHALL be atomic — a `_SAVE_COMPLETE` sentinel file is created only after the last byte is written and the upload returns successfully. A subsequent run's bootstrap MAY discard any partial directory missing the sentinel. The synchronous upload's wall-clock cost is acceptable per Phase 7's measured ~30s/save, and asynchronous uploaders are explicitly out of scope.

## Rationale

vast.ai interruptible offers can preempt the training instance at any time. Local-disk-only checkpoints are lost on preemption. Uploading partials to HF Hub on every save preserves work across instance restarts at the cost of one extra ~30s per save (under 3% of a 6h light-tier run, ~17 saves). Synchronous upload is correct for the actual run shape: smoke runs (~30 min) skip saves entirely; light runs (~6h) save 17 times. An async uploader would add pinned-memory and threading complexity that the spec does not justify. The atomic sentinel prevents resuming from a half-written checkpoint after a crash mid-upload.

## Acceptance Criteria

- [ ] `_save(...)` writes weights and metadata files to `artifacts_dir / f"kdr_{mode}_partial_step{step}"` and only then writes `_SAVE_COMPLETE`
- [ ] After local write completes, the same directory is uploaded synchronously via `huggingface_hub.HfApi.upload_folder` with retry on transient HF errors
- [ ] An interrupted save (process killed mid-`upload_folder`) leaves a partial directory with no `_SAVE_COMPLETE` and is correctly rejected by the resume logic in HLR-0008
- [ ] At end-of-run, the final compressed-tensors checkpoint is uploaded to a separate repo (`pirola/kdr-recovered-{run_id}`) distinct from the partials repo
- [ ] In `bf16` mode with `save_every_n_steps=0` (smoke), the trainer makes zero HF Hub upload calls during training
