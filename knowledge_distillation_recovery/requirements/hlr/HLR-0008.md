## Description

The container bootstrap script (`docker/bootstrap.sh`) SHALL compute a deterministic `run_id` as a stable hash over the tuple `(canonicalised YAML config, student HF revision SHA, mode)`. Before invoking training, the bootstrap SHALL query the HF Hub partials repo `pirola/kdr-partials-{run_id}` for the directory with the highest step number whose `_SAVE_COMPLETE` sentinel is present, download it if any exists, and pass `--resume-from <local_path>` to the trainer. A run with the same `(config, student, mode)` triple on a different vast.ai instance SHALL therefore resume seamlessly. The bootstrap SHALL also `pip install` Zyphra's transformers fork (`Zyphra/transformers@zaya1`) over the base image's stock transformers before training — required for ZAYA1's `ZayaForCausalLM` and harmless on other models.

## Rationale

Vast.ai instances are ephemeral: an interruptible offer dies without warning. Without a deterministic `run_id` derived from inputs, the bootstrap cannot tell "this is a resume" from "this is a new run with similar config." Hashing the canonicalised YAML (post-load Pydantic dump) guarantees that whitespace and comment changes do not invalidate a resume, while semantic config changes do. The Zyphra fork install is necessary because the base GHCR image is shared with `max_quality` and ships stock transformers; ZAYA1 cannot load without the fork.

## Acceptance Criteria

- [ ] `bootstrap.sh` computes `run_id = sha256(canonical_yaml || NUL || student_sha || NUL || mode)[:16]` where `NUL` is the literal byte `\x00` used as a field separator to prevent length-collision attacks across variable-length YAML dumps
- [ ] Bootstrap queries `pirola/kdr-partials-{run_id}` via `huggingface_hub.HfApi.list_repo_tree` and selects the highest `step{N}/` whose `_SAVE_COMPLETE` is present
- [ ] If no partial exists, training starts from step 0; the bootstrap logs this clearly (`"no resume found for run_id={...}, starting fresh"`)
- [ ] If a partial exists but is missing `_SAVE_COMPLETE`, it is skipped and the next-highest valid one is selected
- [ ] Zyphra's transformers fork is installed via `pip install --upgrade --force-reinstall "transformers @ git+https://github.com/Zyphra/transformers.git@zaya1"` BEFORE the trainer is invoked
- [ ] Bootstrap fails fast (non-zero exit) on missing HF_TOKEN, missing CACHE_MOUNT, or missing `STUDENT_REPO` env vars
