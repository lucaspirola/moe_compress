## Description

`kdr.quant.specs.WeightQuantSpec` is a Pydantic v2 strict model with `model_config = ConfigDict(extra='forbid', strict=True)` and the same field set as `KVQuantSpec` (`bits`, `format`, `granularity`, `transform`), applicable to weight quantization. As with `KVQuantSpec`, `transform` in v0 accepts only `Literal['none']`.

## Rationale

Same shape as `KVQuantSpec` to enable shared validation and `config_map` logic. Explicit `extra='forbid'` ensures unknown YAML fields are rejected, not silently ignored — Pydantic's `strict=True` alone does NOT imply `extra='forbid'`.

## Acceptance Criteria

- [ ] `model_config` literally contains `extra='forbid'` and `strict=True`
- [ ] All four fields (`bits`, `format`, `granularity`, `transform`) are required (no defaults)
- [ ] Symmetric 4-bit weights are expressible (`bits=4, format='int', granularity='channel', transform='none'`)
- [ ] Pydantic raises `ValidationError` on a YAML containing an unknown field (e.g., `nbits` instead of `bits`)
