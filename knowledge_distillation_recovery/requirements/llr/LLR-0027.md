## Description

On every `save_every_n_steps` boundary, the trainer writes a partial checkpoint to `artifacts_dir / f'kdr_{mode}_partial_step{step}'` using the same `save_kdr_artifact` path used for final saves.

## Rationale

One save path simplifies maintenance and avoids partial/final divergence.

## Acceptance Criteria

- [ ] Partial dir name embeds both mode and step
- [ ] Partial dir is byte-identical-shape to final dir
