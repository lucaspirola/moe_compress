## Description

`kdr.quant.factory.make_backend(spec) -> QuantBackend` selects ModelOpt vs Native per quantizer based on `feature_matrix.SUPPORTED_QUANTS`.

## Rationale

Allows mixing backends (e.g., NVFP4 weight via ModelOpt + INT3 KV via Native) on the same model.

## Acceptance Criteria

- [ ] Returns ModelOptBackend when the (target, format, bits) tuple is in the matrix; NativeBackend otherwise
- [ ] Test exercises a YAML that mixes (NVFP4 weight via modelopt) with (INT2 V via native)
