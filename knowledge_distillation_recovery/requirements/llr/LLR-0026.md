## Description

ZAYA1 adapter declares `required_attn_implementation() -> 'eager'` (or 'sdpa', whichever the Zyphra fork supports for hookability — to be confirmed at instantiation).

## Rationale

CCA's convolutional downprojector likely doesn't compose with flash-attn fused K/V projection; KV-quant hooks need post-projection K/V exposed.

## Acceptance Criteria

- [ ] Adapter's `required_attn_implementation()` returns one of `'eager'` or `'sdpa'`; flash-attn is rejected
- [ ] Adapter passes `attn_implementation=adapter.required_attn_implementation()` directly to `from_pretrained` for both teacher and student loaders. The trainer does NOT rely on `model.config.attn_implementation` being populated post-load (some HF versions do not store it on the config) — instead the assertion is a constructor-time invariant: `from_pretrained` is called with the explicit attn_implementation and any failure surfaces as a HF-side error
- [ ] A unit test confirms the adapter loads both teacher and student with the declared attn_implementation and that no flash-attn module is present in the resulting model graph
