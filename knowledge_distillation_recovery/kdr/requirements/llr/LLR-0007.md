## Description

In `kdr.training.loop`, branch on `cfg.mode`: if `da_qad`, route the YAML's `quant` block through `quant.factory.partition_and_dispatch(...)` which calls each routed backend's `apply_quant(model, quant_block_subset)` exactly once (per LLR-0013 / LLR-0017) before constructing the optimizer; if `bf16`, skip the entire factory dispatch.

## Rationale

Single-branch dispatch keeps the rest of the loop mode-invariant.

## Acceptance Criteria

- [ ] Exactly one if/else branch on `cfg.mode`
- [ ] In bf16 mode there are zero modelopt calls
- [ ] In da_qad mode `mtq.quantize` is called exactly once before optimizer construction
- [ ] **In da_qad mode the `partition_and_dispatch` call (which transitively invokes `mtq.quantize`) SHALL occur INSIDE the `activate_zero3_init` context that is still open from the student load — i.e. step 4 of LLR-0048's required call order. The context is exited only AFTER `partition_and_dispatch` returns; `accelerator.prepare(...)` is called OUTSIDE the context (step 6 of LLR-0048). Failing this nesting causes ZeRO-3 to silently no-op on the fake-quant-wrapped modules.**
