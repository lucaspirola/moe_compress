## Description

### Module location and entry point

`kdr/tools/kdr_to_gguf.py` is a new module containing:

- A module-level `try: import gguf; except ImportError` guard whose
  except branch raises `ImportError("kdr_to_gguf requires the gguf
  package; install it with `pip install gguf`")` so the failure mode
  is actionable.
- A `_FORMAT_TO_GGUF_TYPE: dict[str, gguf.GGMLQuantizationType]`
  table mapping `"iq2_xs" -> GGMLQuantizationType.IQ2_XS`,
  `"q3_k" -> GGMLQuantizationType.Q3_K`,
  `"iq4_xs" -> GGMLQuantizationType.IQ4_XS`,
  `"q5_k" -> GGMLQuantizationType.Q5_K`. The Profile-7.2 supported
  set only.
- A function `convert(kdr_dir: Path, output_path: Path) -> None`
  that orchestrates: read config + safetensors → encode tensors →
  write GGUF.
- A `main()` function that wraps `argparse`, dispatches to `convert`,
  and handles top-level errors with `sys.exit(1)` + stderr message.
- A `__main__` block calling `main()`.

### `convert` algorithm

```
1. cfg = json.loads((kdr_dir / "config.json").read_text())
2. qcfg = cfg["quantization_config"]
3. config_groups = qcfg["config_groups"]   # dict[str, group]
4. ignore_patterns = qcfg.get("ignore", [])
5. arch = _arch_id(cfg.get("architectures", ["Model"])[0])
6. writer = gguf.GGUFWriter(str(output_path), arch)
7. writer.add_name(cfg.get("_name_or_path", "kdr-recovered"))
8. _add_hparams(writer, cfg)
9. writer.add_kv("kdr.source_dir", str(kdr_dir.absolute()))
10. for tensor_name, tensor in _iter_safetensors(kdr_dir):
       gguf_name = _to_gguf_tensor_name(tensor_name)
       group = _match_group(tensor_name, config_groups)
       if group is not None:
           fmt = group["weights"]["type-or-fmt"]   # see note below
           gguf_type = _FORMAT_TO_GGUF_TYPE[fmt]
           writer.add_tensor(gguf_name, tensor, raw_dtype=gguf_type)
       elif _matches_ignore(tensor_name, ignore_patterns):
           writer.add_tensor(gguf_name, tensor.astype("float16"))
       else:
           raise RuntimeError(
               f"kdr_to_gguf: tensor {tensor_name!r} matched no "
               f"config_groups.targets pattern and is not in the "
               f"ignore list — refusing to silently fall back to F16."
           )
11. writer.write_header_to_file()
12. writer.write_kv_data_to_file()
13. writer.write_tensors_to_file()
14. writer.close()
```

Per-tensor matching uses substring semantics on `tensor_name`
against each `group["targets"]` entry — same as
`NativeBackend._is_carved_out` (LLR-0013).

#### Implementation Notes: `"Linear"` catch-all

`_match_group` recognises the literal pattern `"Linear"` as a
catch-all that matches every tensor name in the safetensors stream.
This is the consumer-side mirror of the v0 Uniform-config path in
LLR-0056, where `_build_quantization_config` emits
`targets: ["Linear"]` to mean "every `nn.Linear` weight tensor".
The safetensors file only contains weight tensors at this stage,
so treating the literal `"Linear"` as a wildcard is equivalent.
All other strings remain plain substring patterns.

### Helper functions

- `_arch_id(architectures_entry: str) -> str` — lowercase, strip
  `"ForCausalLM"` / `"Model"` suffixes. `"ZayaForCausalLM"` → `"zaya"`;
  `"LlamaForCausalLM"` → `"llama"`.
- `_iter_safetensors(kdr_dir: Path) -> Iterator[tuple[str, np.ndarray]]` —
  walks every `*.safetensors` file in `kdr_dir`, yields each tensor.
  Uses `safetensors.numpy.load_file`.
- `_match_group(tensor_name: str, config_groups: dict) -> dict | None` —
  iterates `config_groups.values()` (insertion order); returns the
  group whose `targets` list contains any substring of `tensor_name`,
  else None.
- `_matches_ignore(tensor_name: str, ignore_patterns: list[str]) -> bool` —
  same substring contract as `_match_group`.
- `_to_gguf_tensor_name(hf_name: str) -> str` — pure name translation
  only. Maps `"model.embed_tokens.weight"` → `"token_embd.weight"`
  and `"lm_head.weight"` → `"token_embd.weight"`. All other names
  pass through unchanged. The helper does NOT perform dedup — that
  responsibility lives in `convert()` (see the "Tied embed/lm_head
  handling" section below). The converter is a thin Profile-J layer;
  full HF-→-GGUF naming is the responsibility of llama.cpp's
  convert script when run on the kdr artifact later, if needed.
- `_add_hparams(writer: gguf.GGUFWriter, cfg: dict) -> None` — emits
  the common hparams (`hidden_size` → `<arch>.embedding_length`,
  `num_attention_heads` → `<arch>.attention.head_count`, etc.) via
  the `gguf` library's `Writer.add_*` setters. Phase 7.2 only
  requires presence of `general.architecture` and `kdr.source_dir`
  metadata; full hparam wiring is a Phase-7.3 concern.

### Tied embed/lm_head handling

Detect by inspecting `cfg.get("tie_word_embeddings", False)`. If True,
the converter SHALL always emit the embedding once under the GGUF name
`token_embd.weight`, taking its tensor from
`model.embed_tokens.weight` if that name is present in the
safetensors stream, falling back to `lm_head.weight` otherwise. The
`lm_head.weight` tensor is ALWAYS skipped when `tie_word_embeddings`
is True (it never reaches the GGUF writer as a second emission). The
order in which the two names appear in the safetensors stream does
not affect the output — emission is deterministic.

### Error handling

- Missing `config.json`: `FileNotFoundError` propagates with a clear
  path.
- Missing `quantization_config`: `KeyError` propagates; the GGUF
  converter does not apply outside the compressed-tensors save path.
- Missing `safetensors` files in `kdr_dir`: `RuntimeError` with
  message naming the directory and listing what was found.
- `gguf` not installed at import time: `ImportError` with the
  `pip install gguf` hint (see Description).

## Rationale

The algorithm is a single pass over the safetensors stream; group
matching is O(N·M) in (tensors × groups), both bounded by Profile-J
at ~600 tensors and 4 groups. No special data structures are needed.

The choice to keep `_to_gguf_tensor_name` minimal (only the
embed/lm_head tie) avoids reimplementing llama.cpp's full HF
mapping table — that table is large, evolves with model
architectures, and is the responsibility of llama.cpp's
`convert_hf_to_gguf.py` if a fuller conversion is needed.
Phase 7.2's acceptance gate is "loads in llama.cpp at usable
context length"; a llama.cpp-side post-processing pass on the kdr-
emitted GGUF is acceptable if it lands an in-shape file.

`writer.add_tensor(..., raw_dtype=...)` is the gguf library's path
for emitting a pre-encoded tensor at the target quant type. The
library handles the byte-packing internally based on the
`GGMLQuantizationType` enum.

## Acceptance Criteria

- [ ] `kdr/tools/kdr_to_gguf.py` exists with `main()` and `convert()`
      module-level functions plus a `if __name__ == "__main__":
      main()` guard
- [ ] Module-level import of `gguf` is guarded by `try/except
      ImportError` raising a message containing `pip install gguf`
- [ ] `_FORMAT_TO_GGUF_TYPE` maps exactly the four Profile-J formats
      (iq2_xs, q3_k, iq4_xs, q5_k) to the corresponding
      `gguf.GGMLQuantizationType` members
- [ ] A pytest test stubs the `gguf` module (via a mock module
      object on `sys.modules`) and asserts that
      `_FORMAT_TO_GGUF_TYPE` resolves the four supported formats
- [ ] A pytest test mocks `gguf.GGUFWriter` and invokes `convert()`
      on a stand-in kdr-dir with a 2-group config + a 2-tensor
      safetensors file, asserting that `add_tensor` is called
      exactly twice with the expected `raw_dtype` (or numpy dtype
      for the F16 carve-out path) for each tensor
- [ ] A pytest test mocks `gguf.GGUFWriter` and invokes `convert()`
      on a kdr-dir whose safetensors contains a tensor named
      `mystery_tensor` matching no group and not in `ignore`;
      asserts `RuntimeError` containing the literal `mystery_tensor`
- [ ] `_arch_id("Zaya1ForCausalLM")` returns `"zaya1"`;
      `_arch_id("LlamaForCausalLM")` returns `"llama"`
- [ ] `_match_group` returns the first matching group in insertion
      order; a pytest test confirms first-match-wins behaviour
- [ ] `_to_gguf_tensor_name("model.embed_tokens.weight")` returns
      `"token_embd.weight"`
- [ ] For a `cfg` with `tie_word_embeddings: True`, only one of
      `lm_head.weight` / `model.embed_tokens.weight` reaches the
      GGUF writer (the first encountered)
- [ ] The CLI (`main()`) accepts `--kdr-dir` and `--output` and
      calls `convert(...)` with both as `Path` objects
- [ ] A pytest test confirms `main(["--kdr-dir", "x", "--output",
      "y.gguf"])` calls `convert(Path("x"), Path("y.gguf"))` once
      (via mock)
- [ ] A pytest test exercises the "gguf not installed" path by
      removing `gguf` from `sys.modules` and replacing it with a
      sentinel that triggers `ImportError` on import; reloading
      `kdr.tools.kdr_to_gguf` raises `ImportError` whose message
      contains the literal substring `pip install gguf`
- [ ] The mock-based `convert()` test additionally asserts that
      the `GGUFWriter` mock receives the architecture string as
      either the second positional constructor argument (the gguf
      library's standard path) OR via an explicit
      `add_architecture(...)` / `add_kv("general.architecture", ...)`
      call — whichever the implementation uses — AND receives at
      least one call setting `kdr.source_dir` (via
      `add_kv("kdr.source_dir", ...)`)
- [ ] `_match_group` recognises the literal pattern `"Linear"` as a
      catch-all that matches every tensor name (mirrors the v0
      Uniform-path emission from LLR-0056 where
      `_build_quantization_config` writes `targets: ["Linear"]`).
- [ ] Tied-embedding dedup contract: when both
      `model.embed_tokens.weight` and `lm_head.weight` appear in the
      safetensors stream and `tie_word_embeddings: True`, the GGUF
      writer receives EXACTLY ONE tensor named `token_embd.weight`
      regardless of which name appeared first in the stream
      (deterministic single output, not "first encountered")
