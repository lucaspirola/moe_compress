## Description

Wire `compressed_tensors.converters.ModelOptNvfp4Converter` (or the equivalent class for FP8/INT) inside `ModelOptBackend.save`. Runtime-resolve the converter class via `feature_matrix` so unsupported formats fall through to a clear error.

## Rationale

Avoids emitting modelopt-internal `mto.save` checkpoints; goes straight to the HF-portable format.

## Acceptance Criteria

- [ ] NVFP4 path verified end-to-end on a tiny model
- [ ] Unsupported format raises a clear error naming the missing converter
