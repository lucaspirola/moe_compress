## Description

At end-of-training, upload the final compressed-tensors checkpoint to `pirola/kdr-recovered-{run_id}` (a separate HF Hub repo from the partials repo) and log the URL.

## Rationale

Final and partial repos are separate so partial pruning policy can be aggressive without touching the final artifact.

## Acceptance Criteria

- [ ] Upload happens after the last `_save` returns successfully
- [ ] Logged URL is the actual HF Hub model page
