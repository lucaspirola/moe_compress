## Description

Inside the partial directory, `_SAVE_COMPLETE` is the LAST file written; its presence is the signal that all weight shards and metadata files are intact.

## Rationale

Atomic save invariant: if `_SAVE_COMPLETE` exists, every other file in the dir is committed; otherwise treat the dir as garbage.

## Acceptance Criteria

- [ ] Sentinel is empty (zero bytes) — its presence alone is the signal; a unit test asserts `os.path.getsize(partial_dir / '_SAVE_COMPLETE') == 0` after a successful save
- [ ] Sentinel is the LAST file written (verified by inspecting mtime ordering in a unit test that creates a partial save under a freezegun-controlled clock)
- [ ] Resume logic in HLR-0008 ignores partial dirs lacking it (cross-LLR; verified by LLR-0033's tests, not duplicated here)
