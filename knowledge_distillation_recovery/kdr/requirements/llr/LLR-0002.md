## Description

Implement `_get_kld_loss_fn(temperature)` as a module-level cache returning `LogitsDistillationLoss(temperature, reduction='batchmean')` keyed by temperature.

## Rationale

Avoids constructing the loss object per microbatch — a measurable hot-path cost on the inner loop.

## Acceptance Criteria

- [ ] Cache hits return the same instance
- [ ] Cache key is the float temperature
- [ ] Lazy modelopt import (deferred until first call) so kdr is importable in environments without modelopt
- [ ] Cache access is thread-safe under DataLoader worker concurrency: either the cache is populated eagerly at first call from rank 0 before workers fork, or the cache uses a `threading.Lock` around insert. A unit test constructs two threads racing to populate distinct keys and asserts no exceptions and equal-by-id results for matching keys.
