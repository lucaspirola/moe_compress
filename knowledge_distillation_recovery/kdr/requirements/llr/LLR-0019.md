## Description

If `input_student_dir / 'compressed_metadata.json'` exists, copy it byte-equal into the output directory; if absent, do not create a stub.

## Rationale

MoE-factored topology is structural metadata orthogonal to quantization; preserving it byte-equal avoids schema drift; omitting it for non-factored models avoids fake-empty-stub bugs.

## Acceptance Criteria

- [ ] Byte-equality verified for an input checkpoint that has the file
- [ ] Output for an input lacking the file also lacks it
