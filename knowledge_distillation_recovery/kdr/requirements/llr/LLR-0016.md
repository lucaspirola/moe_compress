## Description

`kdr.quant.modelopt_backend.feature_matrix.SUPPORTED_QUANTS: dict[tuple[Target, Format], list[int]]` declares which `(target, format)` tuples and which bit widths modelopt currently supports. `Target = Literal["weight", "kv_key", "kv_value"]` (underscores, not hyphens — these are Python identifiers consumed by the factory partitioner). `Format` is the same Literal as `kdr.quant.specs.Format`.

## Rationale

Bumping modelopt requires only adding/removing rows — the training code never branches on modelopt version.

## Acceptance Criteria

- [ ] At least the entries for FP8/NVFP4 weight + FP8/NVFP4 KV are populated for the v0 modelopt pin
- [ ] factory.py routes any tuple absent from this matrix to NativeBackend
