## Description

Implement `kdr.kd_loss.forward_kld_loss(student_logits, teacher_logits, temperature=1.0)` that reshapes both tensors to `[B*T, V]`, upcasts each to fp32, and delegates the softmax+KL to `modelopt.torch.distill.LogitsDistillationLoss(temperature, reduction='batchmean')`.

## Rationale

Direct port from `structural_recovery/distillation.py:43-86`. Bit-equal numerical parity with the existing implementation is verified by Phase 3a.

## Acceptance Criteria

- [ ] Function signature is `forward_kld_loss(student_logits, teacher_logits, temperature: float = 1.0) -> Tensor`
- [ ] fp32 upcast happens before softmax
- [ ] Returns a 0-d Tensor
- [ ] Hypothesis test asserts bit-equal output to structural_recovery's version on 100 random inputs and 3 temperatures
- [ ] **Forward-KL direction is asserted explicitly**: `LogitsDistillationLoss.forward(...)` is invoked with `student_logits` as the FIRST positional argument (Q, the predicted distribution) and `teacher_logits` as the SECOND (P, the target distribution) — modelopt's API takes the student first, then the teacher. The loss internally computes `KLD(p_teacher || p_student)` (forward KL with teacher as target). A unit test confirms argument order by feeding asymmetric distributions and checking the resulting loss matches an analytically computed forward-KL value, NOT reverse-KL
