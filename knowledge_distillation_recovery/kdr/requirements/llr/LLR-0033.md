## Description

Bootstrap calls `huggingface_hub.HfApi().list_repo_tree('pirola/kdr-partials-{run_id}', repo_type='model')`, filters to entries matching `kdr_*_partial_step*` whose `_SAVE_COMPLETE` is present, and selects the highest step.

## Rationale

Single source of truth for resume state lives on HF Hub.

## Acceptance Criteria

- [ ] A repo with no matching dirs returns None and the trainer starts from step 0
- [ ] A repo with a partial missing `_SAVE_COMPLETE` is skipped without crashing
