## Description

`kdr.quant.native_backend.ste_simulators` provides pure-function STE
simulators that NativeBackend uses to fake-quantize weights and KV
tensors that modelopt's installed version doesn't ship a kernel for.
Two families:

**Uniform INT simulators (any axis):**

- `int_quant_ste(x, *, bits, axis)` — symmetric integer quant with
  per-slice scale along `axis`. Covers INT2, INT3 (and any other
  uniform width admitted by `bits`).
- `mxfp4_kv_ste(x, *, axis)` — MXFP4 KV simulator (per-token / per-channel
  depending on the consumer's axis choice).

**GGUF codebook simulators (Profile-J formats, `axis=-1`):**

- `iq2_xs_quant_ste(x, *, axis)` — IQ2_XS. Super-block of 256 with
  block-32 internal grouping. Uses both the IQ2_XS magnitude grid AND
  the IQ2_XS sign codebook (from `gguf_codebooks.py`, LLR-0053); a
  magnitude-only argmin is insufficient and would flip negative values.
- `q3_k_quant_ste(x, *, axis)` — Q3_K. Super-block of 256 with 16
  sub-blocks of 16; FP16 super-scale + 6-bit sub-block scales + 3-bit
  quants.
- `iq4_xs_quant_ste(x, *, axis)` — IQ4_XS. Super-block of 256, block-32
  internal grouping, 4-bit quants over the non-linear `KVALUES_IQ4NL`
  codebook plus an FP16 super-scale.
- `q5_k_quant_ste(x, *, axis)` — Q5_K. Super-block of 256 with 8
  sub-blocks of 32; FP16 d/dmin + 6-bit scales + 6-bit mins + 5-bit
  quants.

Common contract for every simulator:

- Forward equals the codebook-snapped (or quantize-then-dequantize)
  value of `x`.
- Backward is the identity over `x` via the STE pattern
  `return x + (q - x).detach()`.
- Scales / zero-points / codebook indices are computed *per-slice*
  along the function's `axis` argument (one scale per index of `axis`,
  shared across all other axes) — except for the GGUF super-block STEs,
  where the super-block-of-256 layout along `axis` is itself the
  per-slice unit.
- The four GGUF STEs use `axis=-1` (in-features axis of an `nn.Linear`
  weight, where the ggml super-block-of-256 layout lives). This is
  intentionally asymmetric to `int_quant_ste`'s `axis=0` convention
  (per-output-channel for `[out, in]` weights). The asymmetry is
  documented and must not be normalised.
- For the GGUF STEs, the size of the last axis MUST be a multiple of
  256; otherwise raise `ValueError` (no implicit padding — a partial
  super-block has no defined round-trip).

NativeBackend's `_install_weight_quant` is responsible for routing each
`WeightPatternSpec` to the matching simulator (the parametrization
wiring layer is a separate concern; the simulators themselves are pure
and own no state).

## Rationale

The Native backend exists to fill holes in modelopt's coverage so the
YAML's recipe space is complete. Two distinct hole shapes are covered
here:

1. Uniform INT widths (INT3, INT2) for KV cache and weight quant —
   modelopt ships no kernel for these.
2. Codebook-based GGUF formats (IQ2_XS, Q3_K, IQ4_XS, Q5_K) for
   Profile-J recovery — these are required by the deployment artifact
   (a llama.cpp-loadable GGUF under 4 GB on disk). Forward fidelity is
   load-bearing: the artifact MUST round-trip through
   `tools/kdr_to_gguf.py` and load in llama.cpp; an STE that
   approximates the codebook with linear quantization invalidates the
   recovered weights at deployment time. The simulators therefore
   consume codebook constants from `gguf_codebooks.py` (LLR-0053)
   rather than approximating with INT-N proxies.

The `axis=-1` choice for GGUF STEs mirrors ggml's row-major super-block
layout for weight tensors; using `axis=0` (per-output-channel) would
group elements across super-blocks and break the codebook fit.

The multiple-of-256 hard-error is the simplest correct contract:
ZAYA1-8B's Linear in-features (hidden dim 5120) is a multiple of 256,
and the contract surfaces a malformed config eagerly instead of
silently quantizing a partial super-block at training time.

## Acceptance Criteria

- [ ] Each simulator is a pure function returning
      `x + (q - x).detach()` (STE gradient identity)
- [ ] Hypothesis property tests on every public simulator covering:
      identity-on-forward, dequant-quant idempotence
      (`q(q(x)) == q(x)`), and gradient flow (`d q(x) / d x == 1`)
- [ ] Granularity correctness for KV cases: K per-channel along
      head_dim, V per-token along seq_len
- [ ] All four GGUF STEs use `axis=-1` as the documented default and
      pass `axis=-1` in their property tests
- [ ] Each GGUF STE raises `ValueError` when
      `x.shape[axis] % 256 != 0`
- [ ] IQ2_XS has a sign-preservation property: for inputs whose
      magnitude is representable, `torch.sign(q(x)) == torch.sign(x)`
      everywhere `x != 0`
- [ ] The four GGUF STEs consume codebook constants from
      `kdr.quant.native_backend.gguf_codebooks` (LLR-0053); no codebook
      literals appear inline
