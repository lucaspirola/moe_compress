## Description

`.pre-commit-config.yaml` runs `req scan --strict` on every commit; commits with un-tagged code or broken traceability are rejected.

## Rationale

Closes the loop between code edits and req graph hygiene at commit time.

## Acceptance Criteria

- [ ] Hook fails the commit if `req scan` reports any error
- [ ] Hook configuration is committed to the kdr package's pre-commit config
