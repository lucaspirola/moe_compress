## Description

Immediately after the partial dir's `_SAVE_COMPLETE` is written, the trainer calls `huggingface_hub.HfApi().upload_folder(repo_id=f'pirola/kdr-partials-{run_id}', folder_path=partial_dir, ...)` with retry on transient HF errors.

## Rationale

Synchronous upload keeps the resume contract simple; the ~30s cost is amortised over hours-long runs.

## Acceptance Criteria

- [ ] Upload happens AFTER `_SAVE_COMPLETE` so a crash mid-upload never leaves a non-resumable local checkpoint
- [ ] Retry logic handles HTTP 5xx and connection timeouts up to 3 retries with exponential backoff
