## Description

`kdr.config.CalibrationConfig` is a Pydantic v2 strict model with `model_config = ConfigDict(extra='forbid', strict=True)` and fields `source: str`, `dataset: str`, `seed: int`, `num_sequences: int`, `sequence_length: int`, `subset_weights: dict[str, float]`, optional `ptq_subset_size: int = 256`.

## Rationale

One YAML block; `ptq_subset_size` is the only optional field with a default. Explicit `extra='forbid'` ensures unknown fields (e.g. `ptq_calibration` accidentally added as a sibling block per HLR-0011 AC #4) are rejected, not silently ignored.

## Acceptance Criteria

- [ ] `model_config` literally contains `extra='forbid'` and `strict=True`
- [ ] Pydantic raises `ValidationError` on a YAML missing any required field
- [ ] Pydantic raises `ValidationError` on a YAML containing an unknown field at the same level (e.g., `ptq_calibration` block as a separate sibling)
- [ ] `ptq_subset_size` defaults to 256 when omitted; explicit values in `[1, num_sequences]` are accepted
