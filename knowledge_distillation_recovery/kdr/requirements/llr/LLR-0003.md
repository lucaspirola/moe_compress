## Description

On entry to `forward_kld_loss`, if `student_logits.shape[-1] != teacher_logits.shape[-1]` raise `ValueError` naming both V dimensions and instructing the caller to use a same-tokenizer pair.

## Rationale

Vocab mismatch silently produces nonsense KL values — surfacing a clear error at the boundary prevents days of silent debugging downstream.

## Acceptance Criteria

- [ ] Raises `ValueError` not generic Exception
- [ ] Error message includes both V values
- [ ] Unit test asserts the specific message text
