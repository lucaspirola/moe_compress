## Description

Single-line change to `kdr/src/kdr/eval/quick.py`, line ~115:

Before:
```python
inp = torch.tensor(ids_list, dtype=torch.long).view(n_seqs, seq_len).to(
    accelerator.device
)
```

After:
```python
inp = torch.tensor(
    ids_list, dtype=torch.long, device=accelerator.device
).view(n_seqs, seq_len)
```

The `.view(...)` call is preserved (operates on a contiguous tensor
identically on any device). The `.to(accelerator.device)` is dropped
because the construction now lands directly on the target device.

## Rationale

`torch.tensor(list, device=cuda)` and `torch.tensor(list).to(cuda)`
are numerically equivalent. The latter performs an unnecessary CPU
allocation and a separate H2D copy. Dropping the intermediate is
purely a cleanup; there is no behavioural change.
