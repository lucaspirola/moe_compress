## Description

`kdr.quant.specs.KVQuantSpec` is a Pydantic v2 strict model with `model_config = ConfigDict(extra='forbid', strict=True)` and required fields `bits: int`, `format: Literal['int','fp8','mxfp4','nvfp4']`, `granularity: Literal['tensor','channel','group','block','token']`, `transform: Literal['none']`. The Literal currently has only `none` because Hadamard / FWHT are deferred to v1+ per the plan; v1 SHALL extend the Literal arms and add NativeBackend routing without touching this LLR's structure.

## Rationale

All four fields are explicit; defaults are forbidden so YAML recipes can't drift. Single-value `Literal['none']` keeps v0 honest about what's actually implemented and prevents YAML recipes that look valid but reference unimplemented transforms.

## Acceptance Criteria

- [ ] mypy --strict accepts an exhaustive `match` on all enum arms (only `'none'` for transform in v0)
- [ ] Pydantic raises `ValidationError` on missing field, unknown format, or unknown granularity
- [ ] Pydantic raises `ValidationError` on `transform: 'hadamard'` or `transform: 'fwht'` in v0 (via `Literal['none']` enforcement)
- [ ] `model_config` sets `extra='forbid'` so unknown fields in YAML are rejected, not silently ignored
