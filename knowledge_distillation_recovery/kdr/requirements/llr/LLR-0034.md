## Description

`kdr.cli.train --resume-from <path>` loads weights, optimizer state, scheduler state, and rng state from the partial dir and resumes the training loop at the recorded step.

## Rationale

Resume parity with structural_recovery's `_load_latest_partial` semantics.

## Acceptance Criteria

- [ ] `--resume-from` argument accepts a local directory path
- [ ] Loaded model's weights match the partial's safetensors byte-for-byte (sanity check)
- [ ] Step counter resumes at the saved step, not step 0
