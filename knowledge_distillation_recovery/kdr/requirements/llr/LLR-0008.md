## Description

`kdr.cli.train` uses `argparse` (or `tap`/`typer`, TBD) to expose `--config <yaml>`, `--student <path>`, `--mode <bf16|da_qad>`, `--artifacts-dir <path>`, `--resume-from <path>` arguments.

## Rationale

Standardised CLI surface so the docker bootstrap can invoke training reproducibly.

## Acceptance Criteria

- [ ] `--mode` defaults to whatever the YAML specifies and overrides if given on the CLI
- [ ] `--resume-from` is optional and a missing path raises FileNotFoundError
