## Description

`pyproject.toml` sets `[tool.mypy]` with `strict = true`, `disallow_untyped_defs = true`, `warn_unreachable = true`, `warn_unused_ignores = true`, and per-package `ignore_missing_imports = true` overrides for `torch.*`, `accelerate.*`, `transformers.*`, `modelopt.*`, `deepspeed.*`, `bitsandbytes.*`, `compressed_tensors.*`. A `[tool.ruff]` block configures the project's lint set (`E`, `F`, `W`, `I`, `N`, `UP`, `B`, `SIM`, `RUF`).

`disallow_any_explicit` is intentionally NOT enabled (per HLR-0012 AC #1's note): Pydantic v2's `ConfigDict` and `Field(...)` API surface exposes `Any` through TypedDict / Field return types, and flagging those would force per-line `# type: ignore` on every Pydantic field. Phase 2 implementation discovered this incompatibility; the remaining strict knobs (`strict`, `disallow_untyped_defs`, `warn_unreachable`, `warn_unused_ignores`) catch the cases we care about.

## Rationale

Static enforcement of the strict-typing invariant, scoped to what is achievable while using Pydantic v2 as the config validator.

## Acceptance Criteria

- [ ] mypy --strict on `src/kdr` exits 0 in CI
- [ ] ruff check exits 0 in CI
- [ ] CI fails on any new mypy/ruff violation
