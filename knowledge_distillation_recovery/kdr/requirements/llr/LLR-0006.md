## Description

Define `kdr.modes.Mode = Literal['bf16', 'da_qad']` and a `ModeConfig(BaseModel)` Pydantic v2 strict model with field `mode: Mode`.

## Rationale

Strict Literal prevents typo-driven runtime mode bugs (e.g. 'BF16' or 'da_QAD').

## Acceptance Criteria

- [ ] mypy --strict accepts pattern matches with all-arms exhaustiveness
- [ ] Pydantic raises ValidationError on invalid mode value at config load
