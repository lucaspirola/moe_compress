## Description

`kdr.adapters.zaya1_8b.Zaya1Adapter` implements `ModelAdapter` against `Zyphra/ZAYA1-reasoning-base`, importing `ZayaForCausalLM` from Zyphra's transformers fork (or via `trust_remote_code=True` if the fork is unavailable). The adapter's `load_teacher_and_student(...)` SHALL load the teacher first, then the student. Both teacher and student are initialised from the SAME pretrained checkpoint (self-distillation).

**Context ownership (critical):** the adapter does NOT enter or exit the `activate_zero3_init` context. The training loop (LLR-0007) opens the context BEFORE calling `load_teacher_and_student` and keeps it open until AFTER `mtq.quantize` returns (per LLR-0048 steps 1–5). The adapter's only responsibility is to perform both `from_pretrained` calls while the context is already active when `is_zero3(accelerator) == True`; the adapter is context-unaware otherwise.

## Rationale

First validated adapter; concrete proof that the Protocol is right. The teacher-first / student-second order under `_activate_zero3_init` is critical because `HfDeepSpeedConfig` must be installed before the student loads or ZeRO-3 sharding silently no-ops.

## Acceptance Criteria

- [ ] Loads via `transformers.AutoModelForCausalLM.from_pretrained(..., trust_remote_code=True)` if the fork is unavailable, else direct import of the fork's `ZayaForCausalLM`
- [ ] On first instantiation, the adapter logs `len(model.model.layers)` (the operative layer count under the Zyphra fork; the paper's Table I says 40 transformer layers but the HF config says `num_hidden_layers: 80` — the empirical count under the fork is the authority; the test asserts the value is in `{40, 80}` and stores it for downstream LLRs to consume)
- [ ] Both teacher and student are loaded from `Zyphra/ZAYA1-reasoning-base` (self-distillation)
- [ ] Teacher load happens BEFORE student load. Under ZeRO-3 (`is_zero3(accelerator) == True`), BOTH the teacher AND the student `from_pretrained` calls occur INSIDE the `activate_zero3_init` context (per LLR-0048 steps 2 and 3 — the teacher is NOT loaded before the context is entered, which would land it as a full-precision replica on rank 0). When `is_zero3` is False, the context is a no-op and load order alone (teacher-then-student) is enforced
- [ ] After `load_teacher_and_student`, teacher's parameters all have `requires_grad=False`
