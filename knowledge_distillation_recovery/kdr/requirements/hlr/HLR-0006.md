## Description

kdr SHALL define a `ModelAdapter` Protocol that each supported model family implements. The Protocol SHALL declare at minimum: `load_teacher_and_student(...)`, `attention_module_paths(model) -> list[str]` (dotted-path strings to submodules whose K/V outputs receive quantization hooks), `kv_quant_exempt_indices(model) -> list[int]` (layer indices whose K/V are NOT quantized — empty list for pure-attention models like ZAYA1), `fp32_carve_outs(model) -> list[str]` (submodules excluded from weight fake-quant — for ZAYA1 these are `lm_head`, routing softmax, CCA cache state, RMSNorm, residual adds per arxiv:2605.05365 §IV-D), `required_attn_implementation() -> Literal["eager","sdpa"]`, and `router_replay_hook(...)` (pins the student's MoE expert assignments to those produced by the teacher on the same input batch — required for MoE QAD stability per arxiv:2605.05365 §IV-C). Adding a new model family SHALL require ONLY adding a new adapter module — no edits to the training loop, save logic, or QuantBackend code.

## Rationale

The original plan assumed standard HF transformer module names (`*self_attn.k_proj*`). ZAYA1-8B uses Compressed Convolutional Attention (CCA) with custom module paths, custom FP32 carve-outs, and routing behavior that requires teacher-pinned replay during precision drift. Hardcoding any of those into the trainer would prevent kdr from supporting a second model family without surgery. The Protocol form bundles all model-specific knowledge into one file per family; the trainer iterates the Protocol's outputs without knowing what model it has. Path strings (rather than nn.Module class names) survive HF version refactors and let adapters target individual layers explicitly.

## Acceptance Criteria

- [ ] `kdr.adapters.base.ModelAdapter` is a `typing.Protocol`; mypy --strict accepts the ZAYA1 adapter as conforming
- [ ] `kdr.adapters.zaya1_8b` implements all Protocol methods and uses the Zyphra transformers fork (`Zyphra/transformers@zaya1`) or `trust_remote_code=True`
- [ ] The ZAYA1 adapter's `kv_quant_exempt_indices` returns `[]` (ZAYA1 has no SSM layers per Table I of arxiv:2605.05365)
- [ ] The ZAYA1 adapter's `fp32_carve_outs` includes at minimum: `lm_head`, the routing softmax submodules, CCA cache state, RMSNorm modules, and residual-stream addition points
- [ ] `router_replay_hook` is invoked by the training loop on every microbatch when the model is MoE; a unit test verifies that student expert assignments equal teacher expert assignments after the hook fires
- [ ] Adding an adapter for a hypothetical second model family SHALL touch zero files outside `adapters/`
