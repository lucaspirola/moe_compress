## Description

The kdr output checkpoint SHALL be a HuggingFace-loadable directory containing (a) sharded safetensors with weights laid out per the [compressed-tensors](https://huggingface.co/docs/transformers/quantization/compressed_tensors) convention, (b) a `config.json` whose `quantization_config` field carries `quant_method: "compressed-tensors"` plus the per-quantizer schema describing what was applied, and (c) the input student's `compressed_metadata.json` (MoE-factored topology — `per_layer_num_experts`, `factored_layers`, `factored_ranks`, `factored_effective_ranks`) preserved verbatim if present, omitted entirely if absent. The output directory SHALL load via `AutoModelForCausalLM.from_pretrained(path, trust_remote_code=True)` without any custom restore step.

## Rationale

Compressed-tensors is the de facto interchange standard for the modern HF/vLLM/TensorRT-LLM/SGLang ecosystem and is natively loaded by HF transformers. Emitting modelopt-internal `mto.save` checkpoints would lock kdr's output to modelopt's restore path and exclude every other consumer. The MoE-factored metadata is orthogonal to quantization — it's structural information from `max_quality/stage5_final` that downstream loaders use to resize router weights. Preserving it verbatim avoids the schema-extension drift the original plan flirted with. Models that did not go through max_quality (e.g., ZAYA1-8B) SHALL NOT have an empty-stub `compressed_metadata.json`; the file is present only when the input had it.

## Acceptance Criteria

- [ ] `kdr.io.save.save_kdr_artifact(model, output_dir, ...)` writes safetensors in compressed-tensors layout (verified by loading via `AutoModelForCausalLM.from_pretrained` and inspecting submodule classes for `CompressedLinear` or equivalent)
- [ ] `config.json` of the output contains `quantization_config.quant_method == "compressed-tensors"` and a `config_groups` block that mirrors the YAML's `quant` recipe
- [ ] If the input student had `compressed_metadata.json`, the kdr output's copy is byte-equal to it (no fields added, none removed)
- [ ] If the input student had no `compressed_metadata.json`, the kdr output also has none
- [ ] Round-trip eval: loading the kdr output and running WikiText-2 PPL produces a value within float-precision tolerance of the kdr-reported PPL at end of training
