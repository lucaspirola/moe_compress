## Description

The kdr Python codebase SHALL be developed under strict typing and CI-enforced traceability:
- Pydantic v2 strict mode AND `extra='forbid'` for every config model (no implicit type coercion AND no silently ignored unknown fields). The two settings are orthogonal in Pydantic v2 — neither implies the other — so both MUST be set explicitly via `ConfigDict(strict=True, extra='forbid')` on every config class.
- `mypy --strict` (no `Any`, no untyped defs, exhaustive Literal matches) clean in CI.
- `ruff check` clean in CI.
- `hypothesis`-based property tests for every `NativeBackend` simulator.
- Pre-commit hook running `mypy`, `ruff`, and `req scan` (the Phase-0-patched req tool that recognises Python `# REQ:` comments).
- Every concrete code unit SHALL carry a `# REQ: LLR-XXXX` tag pointing at the LLR it implements; `req coverage` SHALL show every LLR has at least one CodeRef.

## Rationale

The `max_quality` and `structural_recovery` packages drifted between paper claims, spec text, and implementation despite careful review. The lessons from those passes — captured in the per-phase code-review-loop memory — show that the issues were not in the substantive logic but in unenforced invariants: untyped configs that silently coerced strings to numbers, dictionary-shaped specs that lacked field validation, untagged code that nobody could trace back to a requirement. Strict typing + req tags catches both classes at commit time rather than at review time.

## Acceptance Criteria

- [ ] `pyproject.toml` configures `[tool.mypy]` with `strict = true`, `disallow_untyped_defs = true`, `warn_unreachable = true`, `warn_unused_ignores = true`. **Note**: `disallow_any_explicit` is intentionally NOT enabled. Pydantic v2's `ConfigDict` and `Field(...)` machinery exposes `Any` through TypedDict / Field return types — flagging those would force per-line `# type: ignore` on every Pydantic field, defeating the readability strict typing is meant to provide. The remaining strict knobs catch the cases we actually care about (untyped function signatures, implicit Any returns, unreachable code, unused ignores)
- [ ] Every Pydantic config class in `kdr.config` has `model_config = ConfigDict(strict=True, extra='forbid')`; verified by a meta-test that walks `kdr.config.__dict__` and asserts the property on every BaseModel subclass
- [ ] `pyproject.toml` configures `[tool.ruff]` with the standard ruff lint rule set the workspace agrees on (TBD per project conventions)
- [ ] `.pre-commit-config.yaml` runs `mypy`, `ruff check`, and `req scan --strict`
- [ ] Every public function in `kdr/quant/native_backend/` has at least one hypothesis property test
- [ ] After Phase 5 lands, `req coverage` reports >=90% LLR-to-CodeRef coverage; the remaining <10% are LLRs that intentionally have no code unit (e.g., infrastructure-only LLRs)
