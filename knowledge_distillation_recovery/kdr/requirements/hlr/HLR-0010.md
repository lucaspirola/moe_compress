## Description

kdr SHALL select the optimizer at runtime based on the YAML field `distillation.optimizer`. Values `"adamw_bnb_8bit"` SHALL select `bitsandbytes.optim.AdamW8bit`; values `"deepspeed_cpu_adam"` SHALL select `deepspeed.ops.adam.DeepSpeedCPUAdam` with `adamw_mode=True`. The bnb path is intended for single-GPU runs where DeepSpeed is not used; the DSCPUAdam path requires ZeRO-3 with optimizer offload to CPU and is intended for multi-GPU light-tier runs. Other optimizer values SHALL raise a clear ValueError at config load.

## Rationale

The bnb-8bit + ZeRO-3 combination is broken — bnb owns its `optim.step` and fights DS's fp32 reduce. The two tiers therefore CANNOT share an optimizer; the dispatch is mandatory. DSCPUAdam offloads fp32 master weights and Adam moments to host RAM, leaving per-GPU memory available for the sharded student + activations. This dispatch is preserved verbatim from `structural_recovery/distillation.py:159–196` because it has been validated in static-readiness audits.

## Acceptance Criteria

- [ ] `kdr.training.optim.build_optimizer(student, dconf)` switches on `dconf["optimizer"]` and returns the correct optimizer instance
- [ ] `"adamw_bnb_8bit"` returns `bitsandbytes.optim.AdamW8bit`
- [ ] `"deepspeed_cpu_adam"` returns `DeepSpeedCPUAdam` with `adamw_mode=True`
- [ ] An invalid `optimizer` value raises `ValueError` with a message naming the unknown value and listing the valid ones
- [ ] In `da_qad` mode the optimizer is constructed AFTER `mtq.quantize` so that fake-quant wrapper params are visible to the optimizer's parameter-group construction
