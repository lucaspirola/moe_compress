## Description

kdr's inline eval SHALL compute WikiText-2 perplexity on a YAML-configurable number of sequences. Under DeepSpeed ZeRO-3 the eval forward pass SHALL be rank-replicated (every rank computes the same loss on the same data; no distributed sampler) so that the collective forward semantics required by ZeRO-3 are preserved. Sample generations (when enabled) SHALL be gathered via `deepspeed.zero.GatheredParameters` so that only rank 0 actually invokes `model.generate`, with the parameters re-sharded on context exit. All eval logging SHALL be rank-0-only.

## Rationale

ZeRO-3 shards parameters across ranks and requires forward passes to be collective: every rank must call `student(input_ids=...)` together or the gather will hang. Rank-replicated eval data (no distributed sampler) is the cheapest way to satisfy this — it costs N× the compute for N ranks but avoids deadlocks and per-rank divergence in eval metrics. `model.generate` is incompatible with ZeRO-3's sharded params during decoding, so `GatheredParameters` is required to materialise a full copy on rank 0 for the duration of generation only. This contract is preserved verbatim from `structural_recovery/eval_quick.py:81–134` because it has been exercised against ZeRO-3 in prior runs.

## Acceptance Criteria

- [ ] `kdr.eval.quick._wikitext2_ppl(student, tokenizer, cfg, accelerator)` makes the forward call from every rank with identical input_ids
- [ ] No `DistributedSampler` is constructed for the eval dataset
- [ ] Sample generations (if enabled in YAML) wrap `student.generate(...)` in `with deepspeed.zero.GatheredParameters(student.parameters(), modifier_rank=0):`
- [ ] All `log.info` calls in the eval module are guarded by `if accelerator.is_main_process:`
- [ ] Eval PPL is logged exactly once per `eval_every_n_steps`
